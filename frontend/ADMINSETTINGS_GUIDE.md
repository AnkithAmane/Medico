# AdminSettings Module - Comprehensive Developer Guide

## 📋 Overview

**AdminSettings.jsx** is a professional System Settings & Role-Based Access Control (RBAC) Module designed for clinic administrators. It provides a comprehensive interface for managing clinic configuration, staff permissions, notifications, and security settings.

**Key Features:**
- Sticky two-column dashboard layout
- Profile & clinic information management
- Advanced notification center with toggles
- RBAC matrix with permission management
- Secure password change in collapse component
- Logo upload with preview
- Master toggle permissions per role
- Loading spinners and visual feedback

---

## 🎨 User Interface Architecture

### Layout Structure
```
┌─────────────────────────────────────────────┐

├──────────────┬──────────────────────────────┤
```

### Two-Column Design Benefits
1. **Space Efficiency**: Sidebar remains sticky while content scrolls
2. **Navigation Clarity**: Active section highlighted with left border
3. **Mobile Friendly**: Sidebar converts to horizontal scroll tabs on mobile
4. **UX Pattern**: Matches industry-standard settings interfaces (Google, Microsoft, AWS)

### 5. **RBACSection Component**
Professional Role-Based Access Control Matrix.

**Features:**
- Bordered table with hover effects
- Rows: 8 system permissions
- Columns: 3 user roles (Admin, Doctor, Receptionist)
- Master toggle at column top (Select All/None)
- Checkboxes for individual permissions
- Bootstrap Tooltips on permission names
- Info box warning about permission changes
- Loading state on save

**Permission Matrix:**
```javascript
RBAC_PERMISSIONS = [
  'View Financials',
  'Edit Patient Records',
  'Manage Staff',
  'View Reports',
  'Delete Appointments',
  'Manage Clinic Info',
  'Change System Settings',
  'View Audit Logs'
];

DEFAULT_PERMISSIONS = {
  Admin: [true, true, true, true, true, true, true, true],     // All permissions
  Doctor: [false, true, false, true, false, false, false, false], // Limited access
  Receptionist: [false, true, false, true, false, false, false, false] // Minimal access
};
```

**Master Toggle Logic:**
```jsx
const toggleAllPermissions = (role) => {
  // If all permissions are enabled → disable all
  // If some/none are enabled → enable all
  const allEnabled = newPermissions[role].every((p) => p);
  newPermissions[role] = newPermissions[role].map(() => !allEnabled);
};
```

**UI Pattern:**
```
         ┌─────────────┬───────────┬─────────┬──────────────┐
         │ Permission  │   Admin   │ Doctor  │ Receptionist │
         ├─────────────┼───────────┼─────────┼──────────────┤
         │             │  ☑ [√]   │  ☐     │     ☑        │
         │ View Fin.   │  Toggle   │         │ All Perms.   │
         │             │  All      │  Toggle │ for Role     │
         ├─────────────┼───────────┼─────────┼──────────────┤
         │ Edit Patients│  ☑       │  ☑     │     ☑        │
         │ Manage Staff │  ☑       │  ☐     │     ☐        │
         └─────────────┴───────────┴─────────┴──────────────┘
```

**Tooltip Integration:**
Each permission has a Tooltip that appears on hover, explaining the permission's scope.

## 🚀 Implementation Example

### Connecting to Backend
```jsx
const handleSave = async () => {
  setSaving(true);
  try {
    const response = await fetch('/api/admin/profile', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(profileData)
    });
    
    const result = await response.json();
    if (result.success) {
      alert('Profile saved successfully!');
    } else {
      alert('Error: ' + result.message);
    }
  } catch (error) {
    console.error('Save error:', error);
    alert('Failed to save settings');
  } finally {
    setSaving(false);
  }
};
```

## 🔐 Security Considerations

1. **Password Fields in Collapse**
   - Hidden by default to prevent accidental exposure
   - Only shows on deliberate user action
   - User must explicitly change password

2. **Disabled Fields (License, Registration)**
   - Read-only to prevent accidental modification
   - Cannot be changed through UI
   - Changes only through database administrator

3. **RBAC Warning**
   - Info box warns that permission changes affect all users immediately
   - Encourages careful review before saving

4. **Backend Validation**
   - All changes should be validated on backend
   - User permissions verified for each setting
   - Audit logs should record all changes
